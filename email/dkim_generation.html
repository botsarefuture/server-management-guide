How to Generate a DKIM Key

1. Install Required Tools:

You’ll need OpenSSL or a similar tool for generating the keys. On most Linux systems, OpenSSL is pre-installed.

2. Generate the Private and Public Keys:

Open your terminal and execute the following command to generate a 2048-bit DKIM key pair:

openssl genrsa -out private.key 2048

Extract the public key from the private key:

openssl rsa -in private.key -pubout -out public.key

3. Format the Public Key for DNS:

Open the public.key file and remove the -----BEGIN PUBLIC KEY----- and -----END PUBLIC KEY----- lines.

Also, remove any line breaks so that the key is a single continuous string.

4. Create a DKIM TXT Record:

Log in to your domain’s DNS management console.

Create a new TXT record with the following format:

Name: selector._domainkey.yourdomain.com (replace selector with your DKIM selector, and yourdomain.com with your domain name).

Type: TXT

Value: v=DKIM1; k=rsa; p=YOUR_PUBLIC_KEY (replace YOUR_PUBLIC_KEY with the formatted public key).

5. Configure Your Mail Server:

Depending on your mail server (e.g., Postfix, Exim, etc.), you need to specify the DKIM selector and the path to the private key.

Example for Postfix with OpenDKIM:

Domain yourdomain.com
Selector selector
KeyFile /path/to/private.key

Ensure Postfix is configured to use OpenDKIM for outbound emails.

6. Test Your DKIM Setup:

Send a test email to a service like Mail-tester or DKIM Validator to ensure your DKIM is correctly configured.
